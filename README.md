# Interpretable Gearbox Fault Detection via Kolmogorov-Arnold Networks

Official code repository for the paper:

> **"Interpretable Gearbox Fault Detection via Kolmogorov-Arnold Networks on Statistical Vibration Features"**

---

## Overview

This repository benchmarks a purely interpretable **Kolmogorov-Arnold Network (KAN)** against seven traditional machine learning classifiers for gearbox fault detection using statistical vibration features. Unlike conventional deep learning approaches that act as black boxes, KANs learn activation functions on their edges using B-splines — making every learned feature relationship directly visualisable and physically interpretable.

---

## Novelty Contributions

| # | Contribution | Script |
|---|---|---|
| **NC1** | First standalone pure KAN applied to tabular statistical features for gearbox fault detection (not a CNN hybrid, not a bearing dataset) | `04_kan_training.py` |
| **NC2** | KAN spline visualisation as an XAI tool: marginal activation curves reveal which features have threshold-like, sigmoid-like, or flat (irrelevant) relationships with fault status | `05_interpretability_and_pruning.py` |
| **NC3** | KAN-driven network pruning via L1 spline norms; surviving features validated by re-running all 7 ML classifiers on the pruned feature set | `05_interpretability_and_pruning.py` |
| **NC4** | First comprehensive benchmark of KAN vs. 7 traditional ML classifiers (DT, RF, SVM, NB, KNN, GBC, LR) + MLP under identical 5-fold CV conditions across 6 window sizes | `03_baseline_ml.py` + `04_kan_training.py` |

## Dataset

The **SpectraQuest Gearbox Fault Diagnosis Dataset** is publicly available at:
- [Open Energy Data Initiative (OEDI)](https://data.openei.org/submissions/623)
- [Kaggle mirror](https://www.kaggle.com/)

The dataset contains vibration recordings from a SpectraQuest gearbox simulator at 20 kHz sampling frequency, with 4 accelerometer channels, under 10 load conditions (0 %–90 %), in both healthy and broken-tooth configurations (20 CSV files total).

Place the raw CSVs under `Gearbox Dataset/Healthy/` and `Gearbox Dataset/Broken Tooth/` before running the pipeline.

## Running the Pipeline

Execute the scripts in order:

```bash
# Step 1 — Validate dataset layout
python 01_data_preparation.py

# Step 2 — Extract statistical features for all 6 window sizes
python 02_feature_extraction.py

# Step 3 — Baseline ML classifiers (DT, RF, SVM, NB, KNN, GBC, LR)
python 03_baseline_ml.py

# Step 4 — KAN + MLP benchmark (saves best KAN model to model/)
python 04_kan_training.py

# Step 5 — XAI analysis: spline visualisation + pruning validation
python 05_interpretability_and_pruning.py
```

All outputs are written to `results/`.

## Key Results Summary

| Model | W=300 Acc (%) | W=600 Acc (%) | W=800 Acc (%) |
|-------|:---:|:---:|:---:|
| KAN   | 98.29 | ~100 | 99.96 |
| MLP   | 93.34 | ~99 | 99.00 |
| SVM   | 99.33 | 100 | 100 |
| LR    | 99.13 | 99.94 | 100 |

**Key finding:** KAN outperforms the equivalent MLP by up to **5 %** at the smallest, most volatile window (W=300), demonstrating that B-spline edges adapt faster to sparse statistical manifolds than fixed ReLU weights. At larger windows KAN matches state-of-the-art classifiers while providing native interpretability.

## XAI Analysis Notes

**NC2 — Spline Activation Visualisation**
After training, `05_interpretability_and_pruning.py` sweeps each of the 40 input features from its minimum to maximum value (holding all others at their mean) and records the model's predicted fault probability. This marginal activation curve shows:
- **Threshold / step-like curves** → feature has a sharp decision boundary (e.g., P2P on Sensor 1 spikes above a threshold at tooth breakage)
- **Sigmoid-like curves** → gradual increase in fault probability
- **Flat / near-zero curves** → feature is uninformative for this window size

**NC3 — Pruning Validation**
Features are ranked by the L1 norm of their first-layer spline weights. Features scoring below 5 % of the maximum importance score are pruned. All 7 ML classifiers + KAN are then re-evaluated on this minimal feature subset under the same 5-fold CV, demonstrating that a small subset of sensors and statistics captures nearly all discriminative information.
